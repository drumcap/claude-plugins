# Transfer Essay Expert - 편입 에세이 전문가

이 스킬은 미국 대학 편입 에세이 작성을 전문적으로 지원한다.
`_data/transfer.tsv`의 지원 학교 정보와 학생의 배경을 바탕으로 최적화된 가이드를 제공한다.

---

## 핵심 내러티브 프레임워크

### 전공 전환 스토리 구조
편입 에세이에서 가장 중요한 내러티브는 **왜 지금 학교/전공에서 새 학교/전공으로 가려 하는가**이다.

**효과적인 전환 내러티브 구조:**
1. **발견**: 현재 프로그램에서 더 깊은 분야의 매력을 발견한 계기
2. **한계 인식**: 현재 프로그램이 그 깊이를 채워주지 못하는 구체적 이유
3. **결단**: 새 학교에서 그 갈증을 해소하겠다는 결심
4. **미래 비전**: 새 학교에서의 구체적 목표와 기여

**사용 금지 클리셰:**
- "저는 어릴 때부터 ~를 좋아했습니다"
- "기술로 세상을 바꾸고 싶습니다"
- "도전을 두려워하지 않습니다"
- 모호한 '열정' 표현 → 구체적 경험으로 대체

## 4가지 에세이 유형별 전문 가이드

### 1. Why Transfer Essay

**목적:** 왜 지금 학교를 떠나는가 + 새 학교에서 무엇을 얻을 것인가

**구조 (650단어 기준):**
```
[Hook - ~80단어]
구체적 순간이나 경험으로 시작. 추상적 선언 금지.

[현재 상황과 한계 - ~120단어]
현재 프로그램의 좋은 점 인정 + 한계 서술
- 비판적이되 감사하는 톤 유지
- 구체적 수업명/상황으로 뒷받침

[깨달음과 전환 - ~120단어]
새 분야의 깊이를 발견한 구체적 순간

[새 학교에서의 목표 - ~200단어]
구체적 수업, 교수, 리서치 기회 언급
학교별로 반드시 다르게 작성

[결론 - ~80단어]
현재 경험 + 새 분야 심화 = 미래 고유한 기여
```

**어조 지침:**
- 솔직하고 성숙한 자기인식
- 현재 학교에 대한 원망 금지
- 학문적 갈증과 성장 욕구 강조

---

### 2. Why This School Essay

**목적:** 왜 다른 학교가 아닌 바로 이 학교인가

**핵심 원칙:**
- 반드시 `research/{학교}.md` 내용 기반으로 작성
- 학교별로 완전히 다른 에세이 작성 (복붙 절대 금지)
- 최소 2-3개의 구체적 고유 요소 언급

**구조 (500단어 기준):**
```
[구체적 연결고리 - 1단락]
특정 교수, 랩, 과목, 프로그램 특징 → 지원자 경험과 연결

[프로그램 특징 심화 - 1-2단락]
일반적 명성 말고 구체적 독특한 점

[나와 학교의 시너지 - 1단락]
내가 이 학교에서 어떻게 기여하고 성장할지
```

---

### 3. Why [Major] Essay

**목적:** 해당 전공에 대한 진정한 지적 호기심과 열정 증명

**핵심 원칙:**
- 반드시 구체적 기술/프로젝트/경험으로 뒷받침
- "코딩이 재미있어서" 수준 금지
- 현재 전공 배경이 왜 새 전공에 대한 갈증으로 이어졌는지

**구조 (500단어 기준):**
```
[결정적 경험 Hook - 1단락]
새 분야의 특정 개념/문제에 매료된 순간

[현재 → 목표 전공으로 - 1-2단락]
현재 전공을 하면서 만난 새 분야적 문제들

[구체적 탐구 경험 - 1단락]
스스로 새 분야 개념을 공부하거나 프로젝트를 한 경험

[미래 비전 - 1단락]
새 전공 지식으로 이루고 싶은 구체적 목표
```

---

### 4. Personal Statement

**목적:** 지원자라는 사람 전체를 보여주는 독특한 이야기

**핵심 원칙:**
- 이력서 나열 금지
- 하나의 이야기/경험에 집중
- 성격, 가치관, 사고방식이 자연스럽게 드러나야 함

**구조 (650단어 기준):**
```
[강렬한 Hook - 1단락]
구체적 장면, 대화, 순간으로 시작

[경험 전개 - 2단락]
무슨 일이 있었는가 (스토리텔링)

[배움과 성장 - 1단락]
이 경험에서 무엇을 깨달았는가

[현재의 나와 연결 - 1단락]
이 경험이 편입 결정과 어떻게 연결되는가

[미래 비전 - 1단락]
앞으로 어떤 사람이 되고 싶은가
```

## 에세이 품질 기준

### A급 에세이 특징
- 첫 문장을 읽으면 계속 읽고 싶어진다
- 구체적 이름, 날짜, 상황이 있다
- 이 지원자가 아니면 쓸 수 없는 이야기
- 논리적 흐름이 자연스럽다
- 결론이 강하다

### 흔한 실수 (피해야 할 것)
- "항상 ~에 관심이 있었습니다" → 구체적 경험으로 대체
- "~는 매우 중요합니다" → Show, don't tell
- 수동태 과다 사용
- 단어수 채우기용 반복
- 너무 겸손하거나 너무 자랑스러운 톤

## 국제 편입생 고려사항

### 해외 교육 시스템 설명
- 한국 학교/학과명은 영문 설명 병기
- 수강한 한국 과목명은 영문 설명 추가
- GPA 환산 방식 언급 불필요 (입학처가 처리)

### 언어적 고려사항
- 영어가 제2외국어임을 의식하되, 변명하지 말 것
- 자연스러운 영어를 위해 단순하고 명확한 문장 선호
- 미국 대학 입학사정관 눈높이의 어휘 사용

### 다양성 강점
- 국제 학생의 고유한 관점
- 다국어/다문화 경험
- 해외 교육 경험의 강점 (수학/과학 기초 등)

## 리뷰 프로토콜

에세이 리뷰 시 다음 순서로 피드백:

1. **전체 인상** (3문장): 강점 + 약점 + 전반적 평가
2. **단락별 피드백**: 각 단락 구체적 개선점
3. **상위 3개 수정 우선순위**: 가장 임팩트 있는 변경사항
4. **단어수 및 형식**: 제한 준수 여부
5. **개선 초안 제안**: 약한 문장/단락 재작성 예시 제공

## 버전 관리 규칙

- **절대 기존 파일 덮어쓰기 금지**
- 수정 요청 → 항상 새 버전 번호로 저장
- `status: final` 파일은 수정 불가 (새 버전만 가능)

```
essays/rice/why-transfer-v1.md  ← 초안
essays/rice/why-transfer-v2.md  ← 1차 수정
essays/rice/why-transfer-v3.md  ← 최종본
```
